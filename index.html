Shift-Left Testing Strategy & In-Sprint Automation

The Challenge: Relying solely on mocked unit testing was insufficient to catch all bugs, forcing developers to depend on time-consuming manual testing in the autopush environment prior to release. This approach was inefficient, leading to delayed bug detection, higher costs for resolving issues late in the development cycle, and missed edge cases due to inconsistent manual efforts. Furthermore, the absence of a robust automated framework hindered the achievement of seamless CI/CD workflows.

The Solution: To resolve these inefficiencies, I implemented a Shift-Left strategy leveraging "TIN," a specialized platform designed for hermetic functional testing. This provided an isolated and controlled environment to guarantee consistent and deterministic test results.

  • Hermetic Environment Setup: Built TIN server stacks dedicated to the functional testing of core Manager and Executor components, as well as broader integration and end-to-end (e2e) testing architectures.
  • Comprehensive Coverage: Developed a robust Regression Test Suite encompassing functional, integration, and e2e testing.
  • Pipeline Integration: Integrated the new test suite directly into the CI/CD pipeline to automatically execute tests and trigger auto-deployments based on successful results.

Business Impact:

  • Elimination of Manual Effort: Fully automated testing and deployment pipelines removed the need for slow manual testing and deployment processes.
  • Cost & Time Savings: Bugs were caught significantly earlier in the development cycle, drastically reducing the effort and cost required to resolve them.
  • Unwavering Reliability: Hermetic testing ensured deterministic results, completely eliminating flakiness and variability caused by external dependencies.
  • Accelerated Velocity: Automated testing sped up feedback loops, resulting in faster iterations, improved developer productivity, and a stronger CI/CD workflow with minimal human intervention.

UI Automation Strategy for Hardware-Bound Enterprise Application

The Challenge: The internal enterprise application initially lacked support for UI automation. This posed a significant bottleneck for core hardware automation testing, where the vast majority of test scenarios involved UI interactions. This was particularly tricky because the platform operates as a UI-based application designed specifically for manual execution by QA and BA teams using CRUD semantics, rather than a traditional testing framework.

The Solution: To enable robust UI automation, I implemented a strategy leveraging a specialized hardware orchestration tool. This tool helps teams run tests on custom hardware via API workflows by supplying necessary environment parameters.

  • WebDriver Integration: Engineered Selenium WebDriver tests alongside corresponding workflow configurations.
  • API Orchestration: Configured the enterprise application to invoke the orchestration tool's APIs, successfully passing workflow names and critical parameters.
  • Result Processing: Enabled the orchestration tool to execute the Selenium scripts on the hardware and send the success or failure results back to the main application for further processing.

Business Impact:

  • Broader Adoption: The successful enablement of UI automation led other internal projects to evaluate the application for their automation needs, expanding its usage across teams.
  • Strategic Growth: Upcoming project onboardings will create opportunities to scale resources, ultimately driving growth and cross-functional collaboration.

Testing Strategy Optimization for TP ICAP's tpVOL Platform

Overview: tpVOL is a state-of-the-art currency pair trading platform designed by TP ICAP. It integrates Web, OpenFin, and FIX versions to streamline operations for various user bases across frontend and backend clients.

The Challenge: The project faced significant delays and high costs due to a heavy reliance on a licensed testing tool (VeriFix) and extensive manual efforts for regression testing. The absence of a structured, automated regression framework led to inefficient, error-prone deployment processes that impacted overall delivery quality.

The Solution: Implemented a comprehensive, scalable test strategy grounded in the Test Pyramid.

  • Structured Automation: Prioritized fast, cost-effective automated Unit Tests at the base, Integration Tests in the middle, and selective End-to-End (UI) tests at the top to reduce reliance on slow, manual methods.
  • Modular Framework Design: Developed a robust automation framework supporting Functional, Integration, and End-to-End testing across the platform.
  • Broad Component Coverage: Validated critical system components including the OpenFin Web UI, FIX API, TIBCO Messaging Engine, and GMEX Computational Engines.
  • High Automation Rate: Successfully automated over 95% of regression testing scenarios, improving execution speed and reliability.
  • CI/CD Integration: Integrated the framework with Jenkins to automate deployments and test executions within the CI/CD pipeline, minimizing human error.

Business Impact:

  • Cost Efficiency: Significantly reduced overall testing costs by shifting away from the licensed VeriFix tool to a scalable, automated approach.
  • Accelerated Release Timelines: Optimized deployment workflows enabled much faster updates and reduced overall delivery delays.
  • Enhanced Reliability: Increased test coverage drastically reduced the number of issues reaching production.
  • Scalability: The framework successfully handled increased transaction volumes and platform growth without the need to scale QA resources.
